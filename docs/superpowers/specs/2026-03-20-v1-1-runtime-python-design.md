# Start Your Python v1.1 Design

## Summary

`v1.1` 的目标是把项目从“PyCharm 风格学习器”推进到“更接近真实 PyCharm 的教学型桌面应用”。

本次版本仍然服务 Python 初学者，不做完整 IDE，但会补齐最关键的真实感能力：

- 真实编辑
- 真实运行
- 真实终端交互
- Python 环境缺失时的课程引导

## Product Boundary

### v1.1 要做

1. 中间编辑器中的代码可以真实修改
2. 右上角主 `Run` 只负责进入课程内容态，不直接执行当前编辑器代码
3. 课程内容中的代码块带独立 `Run`，通过系统 Python 执行对应示例
4. 底部终端显示标准输出、错误输出，并支持 `input()` 交互
5. 课程系统继续保留，不丢掉章节树、步骤学习和课程解释
6. 系统中没有可用 Python 时，给出明确提示，并引导到安装课程

### v1.1 不做

1. 内置 Python 运行时
2. 多文件工程运行
3. pip 包管理
4. 调试器、断点、变量监视
5. 完整 PyCharm 级工程能力

## User Flow

### 正常运行路径

1. 用户打开课程 `.py` 文件
2. 用户在中间编辑器中直接修改伪代码
3. 用户点击右上角主 `Run`
4. 中间区域切到课程内容视图
5. 用户点击课程内容中某个示例代码块的 `Run`
6. 应用调用系统 Python 执行该代码块对应的教学示例
7. 底部终端显示输出
8. 如果代码示例使用 `input()`，终端允许用户继续输入

### 未安装 Python 路径

1. 用户点击课程内容中的代码块 `Run`
2. 应用检测不到 `python` 或 `py`
3. 底部终端显示“未检测到 Python”
4. 界面提供“去学习安装 Python”与“重新检测 Python”入口
5. 用户按课程完成安装后重新检测

## UX Design

### 编辑器

- 中间区域从“只读展示”升级为“真实可编辑”
- 保留 Python 风格和 PyCharm 气质
- 当前课程文件作为主要编辑目标
- 主 `Run` 只负责进入课程内容态

### 底部终端

- `Terminal` 不再只是静态文案
- 承担课程代码块的真实执行输出
- 承担 `stdin` 输入
- 支持运行中、等待输入、运行完成、运行失败等状态

### 课程系统

- 左侧课程树继续存在
- 右侧步骤面板继续服务教学
- 右上角主 `Run` 的含义是“开始学习当前课程”
- 中间区域在主 `Run` 后切到课程内容视图
- 课程内容里的代码块各自带真实运行按钮

## Technical Design

### Frontend

- 当前课程代码内容由前端状态单独维护
- 课程源文件和当前编辑中的工作副本需要区分
- 课程内容代码块需要支持“显示代码”和“运行代码”两层模型
- 底部终端需要支持流式输出和输入框

### Tauri Backend

通过 Tauri 命令层实现：

1. 检测系统 Python
2. 启动 Python 子进程
3. 把课程内容中某个代码块对应的运行代码传给子进程
4. 流式返回 stdout / stderr
5. 当 Python 等待输入时，把前端输入写回 stdin
6. 支持停止当前运行会话

### Python Detection

优先顺序建议：

1. `python`
2. `py`

后续版本如有需要，再补“手动指定解释器路径”。

## Delivery Order

1. 真实可编辑编辑器
2. 主 `Run` 与代码块 `Run` 语义拆分
3. 系统 Python 检测与代码块单次运行
4. 底部终端输入输出交互
5. 无 Python 时的安装课程引导
6. 课程系统与真实运行能力的界面收口

## Validation

`v1.1` 完成时至少需要验证：

1. 可以直接修改当前课程代码
2. 点击右上角主 `Run` 后，中间会切到课程内容视图
3. 课程内容中的代码块可以用系统 Python 成功运行
4. 运行错误时能看到 traceback 或错误输出
5. `input()` 场景下可以继续输入
6. 没装 Python 时会出现明确提示和课程引导
